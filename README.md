## Working Pattern

- I start from the real workflow: roles, files, maps, payments, queues, documents, or reports.
- I make the data contract clear: database schema, API response, frontend state, and generated output should match.
- I design tools for repeated use: dashboards need to be scannable, forms need to be predictable, and document output must be verifiable.
- I validate with real artifacts: build output, routes, generated DOCX/PDF/XLSX, map data, runtime logs, and deployment links.
