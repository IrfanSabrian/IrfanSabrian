## How I Build

- Start from the real workflow first: roles, files, maps, queues, documents, reports, or admin tasks.
- Make the data contract clear between database, API, UI state, generated files, and deployment.
- Keep dashboards scannable, forms predictable, and generated documents easy to verify.
- Validate with real outputs: routes, builds, PDFs, DOCX/XLSX files, map data, screenshots, logs, and live links.
