## Current Direction

My recent public and private repositories point to one clear direction: I like building software that turns complicated operational work into usable systems.

- **Spatial and GIS products**: campus maps, infrastructure maps, orderable map products, route logic, floor plans, PostGIS data, ArcGIS/Leaflet interfaces, and SHP/GeoJSON workflows.
- **Operational dashboards**: role-based admin panels, monitoring flows, queue/status tracking, analytics, approval states, and customer/internal portals.
- **AI and document automation**: tender documents, OCR/extraction, spreadsheet-backed workflows, Google Docs/Drive/Sheets integration, PDF/DOCX/XLSX processing, and provider fallback.
- **Desktop and local agent tooling**: Electron/Tauri apps, local runtime control panels, launcher behavior, chat/session UX, and agent routing.

## My Build Pattern

- Start from the real workflow: forms, roles, files, maps, payments, queues, or documents.
- Make the data contract clear: database schema, API response, frontend state, and generated output must line up.
- Design for repeated use: dashboards should be scannable, admin flows should be predictable, and document output should be verifiable.
- Validate with real artifacts: build output, route checks, rendered pages, generated DOCX/PDF/XLSX, map data, and runtime logs.
